### ResourceServer listens on all interfaces (0.0.0.0) — local network audio exposure — cpp:web/resource-server  (`f_20260623202035_4891`, fixed)

### ResourceServer path traversal — any file readable via ../ in /audio/ URL — cpp:web/resource-server  (`f_20260623202035_2022`, fixed)

- path: `src/web/ResourceServer.cpp:96-107`
- evidence: auto *decoded = g_uri_unescape_string(p.c_str(), nullptr); std::string file_path = decoded ? decoded : p; ... std::ifstream file(file_path, std::ios::binary | std::ios::ate);
- suggestion: Resolve the path against a root directory and check that the resolved path starts with the root. Alternatively, require absolute paths (starting with /) and reject any path containing .. — since all audio paths come from FileScanner which returns absolute paths.

### ResourceServer: std::stoll on Range header throws through C callback boundary — cpp:web/resource-server  (`f_20260624165001_7568`, fixed)

- path: `src/web/ResourceServer.cpp:185-187`
- evidence: range_start = static_cast<std::size_t>(std::stoll(std::string(start_str))); range_end = static_cast<std::size_t>(std::stoll(std::string(end_str))); — no try-catch, called inside a SoupServerCallback (C callback).
- suggestion: Wrap in try-catch. If std::stoll throws, respond with 400 Bad Request and return. Or use std::from_chars (already used for volume parsing) which doesn't throw.

### Double-advance on track end when sorting is active — skips every other track — frontend:player-store  (`f_20260623202035_0786`, fixed)

- path: `frontend/src/stores/playerStore.ts`
- evidence: onAudioEvent("ended"): backend auto-advances in IPCHandler::audioEvent("ended"), then the frontend computes the next sorted track and calls playInFolder() again — advancing a second time.
- suggestion: Pick one owner for auto-advance: either disable the backend's auto-advance when sorting is active (check a flag), or let the frontend handle it exclusively and remove the auto-advance from IPCHandler::audioEvent when sorting is active. The sort-branch in onAudioEvent should NOT also call playInFolder — it should just sync state after the backend already advanced.

## MEDIUM

### IPCHandler: std::stoi in state parser throws on corrupt state file — cpp:ipc  (`f_20260624165001_3438`, fixed)

- path: `src/ipc/IPCHandler.cpp:131`
- evidence: auto v = find(key); return v.empty() ? def : std::stoi(v); — if the hand-written JSON parser picks up a non-integer token, std::stoi throws. A corrupted state file at ~/.local/share/treeble/state.json crashes the app at startup.
- suggestion: Wrap in try-catch, return default on parse failure. Or use std::from_chars for all numeric fields (already done for double/volume).
